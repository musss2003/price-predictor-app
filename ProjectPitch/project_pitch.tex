\section{Introduction \& Problem Statement}

\subsection{What is the real-world problem?}
\begin{frame}{What is the real-world problem?}
  \only<1>{
    In the modern housing market, both \textbf{apartment sale prices} and \textbf{rental prices} can vary drastically depending 
    on factors such as location, size, number of rooms, building condition, and surrounding amenities. \newline 
  }
  
  \only<2>{
    However, in many listings, especially on local platforms, price information is often missing, incomplete, or inconsistent.
    This makes it difficult for potential buyers, tenants, and even property owners to assess the true value of an apartment or to set a fair and competitive price.
  }
\end{frame}

\subsection{Why is it important or relevant?}
\begin{frame}{Why is it important or relevant?}
  \only<1>{
    For most people, housing is the \alert{largest financial decision} they will make, yet reliable pricing information is often 
    unavailable. \newline 

    When price data is missing or unrealistic, users have no objective way to determine whether a listing is underpriced, 
    overpriced, or fair. 
  }

  \only<2>{
    A system capable of accurately predicting housing and rental prices would:
    \begin{itemize}
      \item[$\bullet$] Help buyers and renters make more informed decisions.
      \item[$\bullet$] Allow property owners to estimate realistic values for their listings.
      \item[$\bullet$] Enable market analysis, regional comparisons, and price trend visualizations for researchers or real estate agencies.
    \end{itemize}
  }
\end{frame}

\subsection{Our Goal}
\begin{frame}{Our Goal}
  Our goal is to build a predictive model that can estimate both apartment sale prices and rental prices based on publicly 
  \emph{available data}, and integrate the model into an application for ease of use. \pause
  
  \vspace{0.5cm}
  Some concrete examples:
  \begin{enumerate}    
    \item<2-> Interactive map
    \item<3-> Price Fairness Indicator
    \item<4-> Price Prediction Tool for Listings without given prices
  \end{enumerate}
\end{frame}

\subsection{Closer look at the features}
\begin{frame}{Overview of Features}
  We extracted the following features from the real estate listings:

  \begin{itemize}
    \item \textbf{Price}
    \item \textbf{Municipality}
    \item \textbf{Condition}
    \item \textbf{Ad Type}
    \item \textbf{Property Type}
    \item \textbf{Number of Rooms}
    \item \textbf{Area (m²)}
    \item \textbf{Furnishing Level}
    \item \textbf{Floor Level}
    \item \textbf{Heating Type}
  \end{itemize}
\end{frame}

\begin{frame}{Price \& Municipality}
  \textbf{Price:}  
  \begin{itemize}
    \item Continuous numeric feature (float)
    \item Represents the sale or rental price of the apartment in BAM
    \item Serves as the target variable for our regression models
  \end{itemize}

  \pause
  \vspace{0.5cm}

  \textbf{Municipality:}  
  \begin{itemize}
    \item Nominal categorical feature
    \item Indicates where the apartment is located
    \item Values include:
    \begin{itemize}
      \item Sarajevo - Novi Grad
      \item Sarajevo - Centar
      \item Ilidža, Hadžići, etc.
    \end{itemize}
    \item Useful for capturing regional price differences
  \end{itemize}
\end{frame}

\begin{frame}{Condition, Ad Type, Property Type}
  \textbf{Condition:}  
  \begin{itemize}
    \item Nominal categorical
    \item Describes the apartment’s condition (e.g., New Build, Renovated, Under Construction)
    \item Note: One noisy value is "Apartman" (misclassified entry)
  \end{itemize}

  \pause

  \textbf{Ad Type:}  
  \begin{itemize}
    \item Binary categorical
    \item Indicates whether the listing is for \alert{Sale} or \alert{Rent}
    \item Crucial for us, as we aim to build two separate models
  \end{itemize}

  \pause

  \textbf{Property Type:}  
  \begin{itemize}
    \item Binary categorical: Apartment vs. Flat
    \item Values are: \textit{“Apartman”} or \textit{“Stan”}
  \end{itemize}
\end{frame}

\begin{frame}{Rooms, Area, Furnishing Level}
  \textbf{Number of Rooms:}  
  \begin{itemize}
    \item Integer feature
    \item Unlike some countries that use decimal points for half-rooms, Bosnian listings use whole numbers only
  \end{itemize}

  \pause

  \textbf{Area (m²):}  
  \begin{itemize}
    \item Numeric continuous feature
    \item Indicates the size of the apartment in square meters
  \end{itemize}

  \pause

  \textbf{Furnishing Level:}  
  \begin{itemize}
    \item Ordinal categorical feature
    \item Captures whether the apartment is:
    \begin{itemize}
      \item Furnished
      \item Partially furnished
      \item Unfurnished
    \end{itemize}
  \end{itemize}
\end{frame}

\begin{frame}{Floor Level \& Heating Type}
  \textbf{Floor Level:}  
  \begin{itemize}
    \item Mostly integer values (e.g., 0, -2, 3, 7, 11)
    \item Some values are non-standard: \textit{"20+"}
    \item Could be treated as:
    \begin{itemize}
      \item Ordinal (if treated as relative height)
      \item Categorical (if "20+" cannot be normalized)
    \end{itemize}
  \end{itemize}

  \pause

  \textbf{Heating Type:}  
  \begin{itemize}
    \item Nominal categorical
    \item Examples:
    \begin{itemize}
      \item City Central Heating
      \item Electric Heating
      \item Gas Heating
      \item Wood
      \item Other
    \end{itemize}
    \item May affect price and energy efficiency rating
  \end{itemize}
\end{frame}

\subsection{Possible Issues with the data}
\begin{frame}{Observed Limitations}
  \begin{itemize}
    \item Some categorical features (e.g. \texttt{equipment}, \texttt{condition}, \texttt{heating}) have classes with very low representation.
    \begin{itemize}
      \item These can introduce instability during training or bias metrics.
      \item We will decide whether to merge rare classes or drop them depending on the context.
    \end{itemize}
    \item \textbf{Most importantly:} the \texttt{municipality} feature is a poor indicator of location-based price variation.
  \end{itemize}
\end{frame}

\begin{frame}{Why Municipality is Not Sufficient}
  \begin{columns}
    \column{0.55\textwidth}
    \begin{itemize}
      \item For example, \textbf{Novi Grad} covers both:
        \begin{itemize}
          \item high-demand areas near city centers,
          \item and peripheral zones with low real estate value.
        \end{itemize}
      \item Two apartments within Novi Grad may differ drastically in price due to local access to infrastructure and amenities.
      \item Hence, treating \texttt{municipality} as a proxy for "location" leads to poor model generalization.
    \end{itemize}

    \column{0.45\textwidth}
    \centering
    \includegraphics[width=0.9\textwidth]{Images/NoviGradMap.png} % Replace with your image
    \newline
    \scriptsize\textit{Illustration: Disparity in spatial price variation within one municipality.}
  \end{columns}
\end{frame}

\subsection{Possible Solutions}
\begin{frame}{Proposed Solution: Geo-based Enrichment}
  \begin{itemize}
    \item We plan to extract the \textbf{longitude and latitude} of each apartment (either from address parsing or manual geocoding).
    \item With this data, we can:
    \begin{itemize}
      \item Use the \textbf{Overpass API} and \textbf{OpenStreetMap} to query surrounding infrastructure.
      \item Compute features like:
      \begin{itemize}
        \item Number of tram/bus stations within 500m.
        \item Distance to nearest hospital, park, supermarket, etc.
        \item Average distance to \textit{n} key public service locations.
      \end{itemize}
    \end{itemize}
    \item This makes our model more \textbf{location-aware} and enables predictions even in previously unseen areas.
  \end{itemize}
\end{frame}
